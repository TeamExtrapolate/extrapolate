Machine Learning Pipeline

Data Accumulation

Based on the problem and our interpretation of it, we collated data for our solution from the following sources:

1. AICTE
2. Educational Census
3. IKDD (Academic) Data

Preprocessing

Preprocessing involves transforming raw data into a usable format.

Example: We had to adjust the skewness of the predictor variable, salary, by logarithmic standardization of the values since skewed data can have ruinous consequences for our predictive model.

Feature Selection

Attributes that were relevant for the predictive analysis were identified while others including the dependent ones were dropped. These relevant features were then checked for heteroscedasticity and skewness.
 For instance, following were the results obtained from PCA.

Model Building/Selection

Data Split: 80:20 :: Training:Cross Validation

The following models were considered (regressors):

1. SVM
2. Random Forest
3. Ridge Regression
4. Lasso Regression
5. Artificial Neural Network(MLP)

The RMSE values for each model are shown below.

Since the lowest value of the RMSE was observed for Ridge Regression, it was selected and hyper-parameter tuned (GridSearchCV with 3-Fold Cross Validation to determine the best possible values of its parameters) to integrate with the backend working of the API.

Model Application

The model was applied to a sample testing dataset (completely unused) and correspondingly the salaries were predicted. Finally, we plotted a histogram to visualise the distribution of the predicted salaries to ascertain whether the distribution was similar to that of the training set salaries. As a result, the machine learning methodology was validated since the distribution was visibly similar.
